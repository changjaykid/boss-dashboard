# 虛擬貨幣賺錢機 💰

## 核心定位

MAX 交易所多幣種 TWD 現貨交易。
主力為**低買高賣分批模式** — 跌時分批接、漲時分批賣，穩穩賺價差。

**交易標的**：BTC/ETH 固定 + 任意 2 個動能高的幣種（每 6 小時自動掃描）
**操作模式**：低買高賣 + 分批建倉 + 分批出貨
**核心目標**：帳戶本金最大化

---

## 囤幣模式策略

### 買入邏輯（分批低接）
- 將可用資金分成 **5 等份**
- 每跌破一個支撐位 → 買入 1 份
- 支撐位判定：近期低點、整數關卡、BB 下軌、EMA50
- RSI < 30 加碼買（超賣區＝撿便宜）
- FGI < 20 極度恐懼 → 額外加碼信號

### 賣出邏輯（分批獲利）
- 持倉成本以上 +1% → 賣出 20%
- 持倉成本以上 +2% → 再賣 20%
- 持倉成本以上 +3% → 再賣 30%
- 剩餘 30% 用追蹤止盈，讓利潤跑
- RSI > 70 超買 → 加速出貨

### 關鍵原則
- **永遠保留 20% 現金** — 不要 all in，留子彈接更低價
- **不追高** — 已經漲 3% 以上不買
- **均價控制** — 每次買入後重新計算持倉均價

---

## 絕對鐵律（違反任何一條 = 停機檢討）

1. **本金保護第一** — 單筆最大虧損不超過本金的 2%（NT$200）
2. **不確定就不做** — 寧可錯過，不可做錯
3. **虧小賺大** — 停損快狠準，停利讓利潤跑
4. **每筆交易必須有明確理由** — 不是「感覺」，是「因為 A+B+C 所以進場」
5. **日虧上限 NT$1,000** — 觸發立刻停機，當日不再交易
6. **連虧 3 筆停機 1 小時** — 冷靜，重新分析市場
7. **永遠先看大方向，再找小週期進場點**

## 三維決策系統

每次決策必須同時考慮三個維度，缺一不可：

### 維度一：技術面（權重 50%）

**15 分鐘 K 線主力指標：**

| 指標 | 用途 | 參數 |
|------|------|------|
| EMA 8/21 | 趨勢方向 + 黃金/死亡交叉 | 8, 21 |
| RSI(14) | 超買超賣 + 背離 | 14, 超買>70 超賣<30 |
| MACD | 動能確認 + 柱狀圖變化 | 12, 26, 9 |
| 布林通道 | 波動率 + 擠壓突破 | 20, 2σ |
| ATR(14) | 波動幅度 → 計算停損 | 14 |
| 成交量 | 量能確認 | 對比 20 根平均 |
| VWAP | 當日公允價格 | 日內 |

**輔助確認（1H + 4H）：**
- 1H 趨勢方向（EMA 排列、MACD）
- 4H 大趨勢（只看方向，不反著做）
- **規則：15M 進場方向必須與 1H 一致，不逆 4H 大趨勢**

**K 線型態（優先級高）：**
- 吞噬型態（看多/看空吞噬）
- 錘子線 / 吊人線
- 十字星（變盤信號）
- Pin Bar（假突破反轉）
- 連續同向 K 線 ≥3 根（動能強勁）

**圖表型態：**
- 雙頂/雙底
- 頭肩頂/底
- 三角收斂（突破方向跟進）
- 旗形/楔形（趨勢延續）
- 支撐/壓力位（近期高低點、整數關卡）

### 維度二：新聞面（權重 30%）

**必須監控的新聞源：**
- Fed 利率決議 / FOMC 會議
- CPI、PPI、非農等美國經濟數據
- BTC ETF 資金流入流出
- 重大交易所事件（被駭、監管、上下架）
- 巨鯨錢包異動
- 各國加密貨幣法規變動

**新聞觸發規則：**
- 🟢 **重大利多**（ETF 大量流入、Fed 降息）→ 提高做多信心，放寬進場條件
- 🔴 **重大利空**（交易所爆雷、監管打壓）→ 提高做空信心 or 減倉觀望
- ⚪ **無重大新聞** → 純技術面交易
- ⚠️ **即將公布重大數據** → 數據公布前 30 分鐘停止開新倉

**新聞資料來源：**
- 使用 crypto-news skill 的資料（`boss-dashboard/crypto-news.json`）
- Web search 即時查詢重大事件
- `trading/crypto_updater.js` 定期更新

### 維度三：社群情緒（權重 20%）

**監控來源：**
- X (Twitter)：#BTC、#Bitcoin、KOL 發文
- Reddit：r/Bitcoin、r/CryptoCurrency
- Fear & Greed Index（alternative.me）

**情緒信號：**
- 恐懼貪婪指數 < 25（極度恐懼）→ 考慮反向做多（逆勢機會）
- 恐懼貪婪指數 > 75（極度貪婪）→ 考慮減倉或做空
- KOL 集體唱多/唱空 → 小心反轉（逆向指標）
- 討論量暴增 → 波動即將放大，收緊停損

## 進場條件（必須全部滿足）

### 做多（BUY）
1. ✅ 4H 趨勢不是 strong_down
2. ✅ 1H EMA8 > EMA21（1H 看多）
3. ✅ 15M 出現以下至少 2 個信號：
   - EMA8 上穿 EMA21
   - RSI 從 <30 回升到 >35
   - MACD 柱狀圖由負轉正
   - 看多 K 線型態（吞噬、錘子、Pin Bar）
   - 價格突破布林中軌且成交量放大
4. ✅ 成交量 > 20MA 的 1.2 倍（量能確認）
5. ✅ 無重大利空新聞
6. ✅ 風報比 ≥ 2:1

### 做空（SELL）
1. ✅ 4H 趨勢不是 strong_up
2. ✅ 1H EMA8 < EMA21（1H 看空）
3. ✅ 15M 出現以下至少 2 個信號：
   - EMA8 下穿 EMA21
   - RSI 從 >70 回落到 <65
   - MACD 柱狀圖由正轉負
   - 看空 K 線型態（吞噬、吊人、射擊之星）
   - 價格跌破布林中軌且成交量放大
4. ✅ 成交量 > 20MA 的 1.2 倍
5. ✅ 無重大利多新聞
6. ✅ 風報比 ≥ 2:1

### 加分條件（有這些可以提高倉位）
- 恐懼貪婪指數配合方向（做多 + 恐懼 < 30；做空 + 貪婪 > 75）
- 多時間框架完全對齊（4H + 1H + 15M 同向）
- 新聞面支持方向
- 布林通道擠壓後突破

## 出場規則

### 停損（鐵律，不可移動放寬）
- **固定停損**：ATR(14) × 1.0，最大不超過本金 2%（NT$200）
- 進場後立即設定，不可更改
- 如果 ATR 太大導致停損 > NT$200，縮小倉位直到符合

### 停利（動態追蹤，讓利潤跑）
- **第一階段**：盈利達 0.3% → 移動停損到成本價（保本）
- **第二階段**：盈利達 0.6% → 啟動追蹤停損，回撤 0.3% 平倉
- **第三階段**：盈利達 1.0% → 收緊追蹤到回撤 0.2%
- **第四階段**：盈利達 1.5% → 追蹤回撤 0.15%（讓大利潤跑到極致）

### 時間出場
- 持倉超過 2 小時且利潤 < 0.1% → 平倉（資金效率）
- 持倉超過 4 小時 → 無論盈虧強制平倉

### 特殊出場
- 出現反向重大新聞 → 立即平倉
- 恐懼貪婪指數劇烈變動（單日變化 > 15）→ 收緊停損
- 進入重大數據公布前 30 分鐘 → 平倉所有持倉

---

## 倉位管理

### 基礎倉位
- **標準倉位**：NT$2,000（本金的 20%）
- **高信心倉位**：NT$3,000（本金的 30%）— 三維度全部對齊時
- **低信心倉位**：NT$1,000（本金的 10%）— 只有技術面信號時

### 加倉規則
- 持倉盈利 > 0.5% 且趨勢未變 → 可加倉 50% 原始倉位
- 加倉後重新計算整體停損
- 最大同時持倉：1 筆（專注做好每一筆）

### 連虧縮倉
- 連虧 2 筆 → 倉位減半
- 連虧 3 筆 → 停機 1 小時
- 日虧達 NT$500 → 倉位減至最小（NT$1,000）
- 日虧達 NT$1,000 → 停機到隔天

### 連勝加倉（滾動複利）
- 連勝 3 筆 → 標準倉位提高 20%
- 連勝 5 筆 → 標準倉位提高 40%
- 但任何單筆虧損 → 倉位回到基礎水準

## 每次 Cron 執行流程

```
Step 0: 檢查停機狀態（日虧上限、連虧暫停、重大數據前）
Step 1: 取得 MAX API 數據（BTC/TWD ticker、15M/1H/4H K線、order book）
Step 2: 計算技術指標（EMA、RSI、MACD、BB、ATR、VWAP、成交量）
Step 3: 讀取新聞資料（crypto-news.json + 即時 web search 重大事件）
Step 4: 讀取社群情緒（Fear & Greed Index + X/Reddit 關鍵詞）
Step 5: 管理現有持倉（更新追蹤停損、檢查出場條件）
Step 6: 如果無持倉 → 三維度綜合判斷是否進場
Step 7: 如果決定進場 → 計算倉位大小、停損停利、下單
Step 8: 更新 state.json、log.json
Step 9: 更新 money-maker-data.json（主控版）
Step 10: 如果有異常或重大事件 → LINE 通知老闆
```

## MAX 交易所注意事項

- **手續費**：Maker 0.08% / Taker 0.16%（VIP 0）
- 一進一出約吃 0.16-0.32%，小波段利潤容易被吃掉
- **盡量用限價單**（Maker），節省一半手續費
- **最小交易量**：BTC/TWD 最小掛單量需查 getMarkets
- **滑價**：深夜流動性低，大單注意分批
- **凌晨 2-6 點**：流動性最差，考慮降低倉位或暫停
