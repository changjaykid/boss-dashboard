# Binance USDT-M 永續合約自動交易系統 📈

## 核心定位

多空雙向、多策略、自動進化的合約交易機器。
不是一招打天下，是持續找最賺錢的打法。

**交易所**：Binance（USDT-M 永續合約）
**標的**：BTCUSDT（主力），可擴展 ETHUSDT 等
**槓桿**：5x（逐倉 ISOLATED）
**目標**：利潤最大化 × 嚴格風控

---

## 絕對鐵律

1. **每筆必掛止損** — 不裸扛，絕無例外
2. **單筆最大虧損 ≤ 保證金 2%** — 超過就是系統 bug
3. **日虧上限 ≤ 保證金 5%** — 觸發停機當日不再交易
4. **連虧 3 筆暫停 1 小時** — 冷靜重新評估
5. **不凹單** — 錯了就認，快砍
6. **不逆 4H 大趨勢開倉**（除非極端反轉策略）
7. **虧小賺大** — 風報比 ≥ 1.5 才進場

## 多策略矩陣

引擎根據市場狀態自動選擇策略，不同狀態不同打法：

### 策略 1：趨勢突破（BREAKOUT）
- **適用**：BB 擠壓後爆發、盤整區突破
- **進場**：價格突破 BB 上/下軌 + 量能放大 > 1.5x 均量
- **方向**：突破上軌做多、突破下軌做空
- **止損**：BB 中軌
- **止盈**：ATR × 3
- **預期勝率**：40-50%｜**風報比**：1:3+

### 策略 2：均線順勢回踩（TREND_FOLLOW）
- **適用**：明確趨勢中的回調
- **進場**：EMA8/21 金叉後回踩 EMA21 不破 + RSI 回到 40-60 區間
- **方向**：順 EMA 排列方向
- **止損**：EMA21 下方 1 ATR
- **止盈**：前高/前低 或 ATR × 2
- **預期勝率**：50-60%｜**風報比**：1:2

### 策略 3：極端反轉（REVERSAL）
- **適用**：FGI < 15 或 RSI < 25（做多）、FGI > 85 或 RSI > 75（做空）
- **進場**：極端值 + K 線反轉型態確認（吞噬、錘子、pin bar）
- **方向**：逆極端方向
- **止損**：極值點下方/上方 1 ATR
- **止盈**：ATR × 4（讓利潤跑）
- **預期勝率**：35-45%｜**風報比**：1:4+

### 策略 4：動量追擊（MOMENTUM）
- **適用**：15 分鐘內暴漲/暴跌 > 0.5%
- **進場**：動量爆發 + 量能確認 + MACD 同向
- **方向**：順動量方向
- **止損**：爆發起點
- **止盈**：追蹤止盈（見下方）
- **預期勝率**：45-55%｜**風報比**：1:2

### 策略 5：資金費率套利（FUNDING_ARB）
- **適用**：資金費率極端（> 0.05% 或 < -0.05%）
- **進場**：費率過高做空、費率過低做多（吃結算費率）
- **方向**：反費率方向
- **止損**：嚴格窄止損（0.3%）
- **止盈**：費率結算後 + 小幅價格回歸
- **預期勝率**：70%+｜**風報比**：1:1

### 策略 6：新聞事件驅動（NEWS_EVENT）
- **適用**：FOMC、CPI、非農等重大數據公布
- **進場**：數據公布後方向確認，等 5 分鐘避開初始假動作
- **方向**：順數據方向
- **止損**：公布前價格
- **止盈**：ATR × 3
- **預期勝率**：40-50%｜**風報比**：1:3

## 市場狀態辨識

引擎每次決策前先判斷市場狀態，決定啟用哪些策略：

| 狀態 | 判斷條件 | 啟用策略 |
|------|----------|----------|
| **趨勢上漲** | EMA8 > EMA21 > EMA50，ADX > 25 | TREND_FOLLOW, MOMENTUM |
| **趨勢下跌** | EMA8 < EMA21 < EMA50，ADX > 25 | TREND_FOLLOW, MOMENTUM |
| **盤整** | ADX < 20，BB width < 2% | BREAKOUT, FUNDING_ARB |
| **高波動** | ATR > 2x 均值，BB width > 4% | MOMENTUM, NEWS_EVENT |
| **極端情緒** | FGI < 15 或 > 85 | REVERSAL |

## 動態倉位管理

根據信號強度調整倉位大小：

- **高信心**（≥ 4 個信號確認）：保證金的 30%
- **中信心**（3 個信號確認）：保證金的 20%
- **低信心**（2 個信號確認）：保證金的 10%
- **1 個信號以下**：不進場

### 加倉邏輯
- 持倉盈利 > 1%，趨勢仍明確 → 可加倉 50%（總倉不超過 40%）
- 加倉止損移到成本價（保護利潤）

## 追蹤止盈系統（階梯式）

不急著跑，讓利潤最大化：

| 浮盈 | 動作 |
|------|------|
| +0.3% | 啟動追蹤，止損移到成本價（保本） |
| +0.6% | 止損移到 +0.3%（鎖利） |
| +1.0% | 止損移到 +0.6% |
| +1.5% | 止損移到 +1.0% |
| +2.0% | 止損移到 +1.5% |
| +3.0%+ | 止損維持在盈利的 50% |

## 自動復盤優化系統

### 每日復盤（06:00 自動跑）

**1. 績效分析**
- 總勝率、總風報比、夏普比率、最大回撤
- 每個策略獨立績效評分
- 每個時段表現（亞洲/歐洲/美洲盤）

**2. 策略權重調整**
- 連賺 3 筆策略 → 權重 +20%
- 連虧 5 筆策略 → 權重 -50%
- 30 筆後勝率 < 30% 且風報比 < 1.5 → 自動停用
- 停用策略每週重新評估一次（市場可能變了）

**3. 參數自動調優**
- 止損距離：過去 50 筆平均 ATR 校正
- 進場門檻：假信號率 > 60% → 提高門檻
- 持倉時間：分析最佳平倉時機
- 量能門檻：根據近期平均成交量動態調整

**4. 市場適應**
- 偵測市場風格變化（從趨勢轉盤整、從低波動轉高波動）
- 自動切換策略組合
- 記錄「什麼市場狀態 + 什麼策略 = 最賺」

### 每週復盤（週日 06:00）
- 本週 vs 上週績效對比
- 最賺策略 vs 最虧策略
- 優化建議清單
- 更新策略參數

- 🔴 **重大利空**：提高做空信號權重 +30%
- ⚪ **無重大新聞**：純技術面
- ⚠️ **重大事件前 1 小時**：縮小倉位 50%，等數據出來再動

- 結算前 30 分鐘檢查費率
- 費率 > 0.05% → 考慮做空吃費率
- 費率 < -0.05% → 考慮做多吃費率
- 有持倉時：評估費率成本，必要時提前平倉

## 執行流程（每 3 分鐘）

```
1. 抓取市場數據（K線、深度、資金費率）
2. 檢查風控（日虧限制、連虧暫停、持倉時間）
3. 若有持倉 → 管理持倉（追蹤止盈、止損、加倉判斷）
4. 若無持倉 → 辨識市場狀態 → 選擇策略 → 分析信號 → 決策
5. 有信號 → 計算倉位 → 下單 + 掛止損
6. 更新狀態 + 主控版
7. 記錄日誌
```

## 安全機制

- **API 權限**：僅開啟 Futures 交易，不開提幣
- **IP 白名單**：僅信任的 IP 可用
- **最大持倉**：不超過保證金的 40%（含槓桿後名義價值）
- **斷線保護**：市場數據超過 5 分鐘未更新 → 不做新決策
- **異常偵測**：單筆虧損超過預期 2 倍 → 停機通知老闆
