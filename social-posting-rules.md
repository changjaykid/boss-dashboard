# 社群發文規則

## 發文流程 SOP
1. 管家產出草稿，放進 social-data.json 的 `approvalQueue`
2. 老闆在主控板社群專案頁面看到草稿
3. 老闆確認 → 管家發文
4. 老闆不滿意 → 管家重寫一篇新的替換
5. **初期每篇都要老闆審核，未來再開放自動發文**

## 內容風格
- 口吻像真人說話，不要 AI 味、不要文青腔過頭
- **禁止使用「——」符號**
- 說話自然，像在跟朋友聊天或分享經驗
- 可以有個性、有態度，但不要刻意裝腔

## 平台特殊規則
### Threads
- **不放 hashtag**
- 內容簡潔有力，適合滑手機閱讀

## 內容主題（Pillars）
1. 市場觀察：趨勢解讀、產業動態
2. 實戰技巧：行銷操作、廣告投放心得
3. 案例拆解：成功/失敗案例分析
4. 觀點輸出：犀利觀點、逆向思考
5. 幕後分享：工作日常、真實經歷
6. 乾貨炸彈：純乾貨、教學級內容

## 狀態說明
- `pending_approval`：等待老闆審核
- `approved`：老闆確認，準備發文
- `rejected`：老闆不滿意，需重寫
- `published`：已發布
