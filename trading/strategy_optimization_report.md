## 六、建議新增策略方向

### 1. 📊 強化 Mean Reversion（均值回歸）
- **依據**：Crypto mean_revert 是唯一正收益策略類型（+227 TWD, 75% WR）
- **方向**：
  - 開發 Forex 版均值回歸策略（RSI 極端 + Bollinger 回歸，而非趨勢中抄底）
  - 加入成交量確認條件
  - 限制只在 RANGING 市場體制下使用

### 2. 🎯 Session Breakout（時段突破）
- **依據**：目前沒有基於時段的策略
- **方向**：
  - 倫敦開盤突破 (08:00-09:00 UTC)
  - 紐約開盤突破 (13:00-14:00 UTC)
  - 亞洲區間突破
  - 這些有統計優勢，比純技術指標突破更可靠

### 3. 📉 Volatility Contraction Pattern（波動收縮）
- **依據**：多筆「布林極度擠壓(突破在即)」信號被使用但效果差，方向判斷有問題
- **方向**：
  - 不預判方向，等突破後確認再入場
  - ATR 收縮到歷史低點 → 設雙向掛單 → 突破方向跟進
  - 配合成交量放大確認

### 4. 🔄 Multi-Timeframe Divergence（多時間框架背離）
- **依據**：很多虧損交易是「趨勢信號強但 RSI 已經超買/超賣」
- **方向**：
  - 1H 趨勢 + 15M RSI 背離 = 反轉信號
  - 比單純 RSI 極端更可靠
  - 配合 K 線反轉形態確認

### 5. 💰 改良版趨勢跟隨（嚴格過濾）
- **依據**：趨勢跟隨概念沒錯，但執行有嚴重問題
- **方向**：
  - 強制要求 3 個時間框架完全一致才入場
  - RSI 不能在超買/超賣區（避免趨勢末端入場）
  - 最小持倉時間（避免被噪音掃出）
  - **限制每個商品每天最多 2 筆交易**（解決 15M_趨勢跟隨一天做 13 筆 BTC 的問題）

## 七、系統層面問題 & 建議

### 🐛 Bug 修復（優先）

1. **Forex pnl=0 問題**：43 筆交易的 P/L 未從 broker 回收。必須修復 `analyzer_v5.py` 的 SL/TP_BY_BROKER 處理邏輯，查詢 `/api/v1/history/transactions` 取得實際盈虧。

2. **重複平倉記錄**：Crypto log 中同一倉位出現 2 次 CLOSE（BTC SELL entryPrice=2155555.5 和 BTCUSDT SELL entryPrice=67200 各出現 2 次），導致虧損被重複計算。

### 📐 風控改進

3. **Position Sizing 問題**：
   - 趨勢對齊在 US30 上 size 從 0.15 → 0.28 → 0.42 逐步加碼，典型的「虧損加碼」反模式
   - **建議**：固定 position size，不隨信心加碼；或虧損後減半

4. **入場頻率控制**：
   - 15M_趨勢跟隨 2 天做了 24 筆，其中 BTC 13 筆（幾乎每 15 分鐘做一次）
   - **建議**：同一策略+同一商品，每天最多 2-3 筆

5. **信心分數校準**：
   - 多筆 confidence ≥ 0.80 的交易虧損（包括 0.95 的也虧）
   - **建議**：回測信心分數 vs 實際勝率，重新校準閾值

6. **停損寬度**：
   - Crypto trend_follow 平均虧損 -280 TWD vs 平均獲利 +99 TWD
   - **建議**：SL 應更緊（ATR × 1.0 而非 1.2），或用固定百分比上限（0.5%）

### 🧪 策略進化流程

7. **回測 → 實測 gateway 太寬**：V5 系列和新 breakout/reversion 策略全敗，應在回測階段就被淘汰。
   - **建議**：回測要求最低勝率 30% + 風報比 1.5:1 才能進入實測

8. **策略退休標準**：
   - 連續 5 筆虧損 → 自動暫停
   - 20 筆交易後勝率 < 25% 且風報比 < 1.0 → 自動退休
   - 單筆虧損超過日均獲利 3 倍 → 暫停檢討

## 九、總結

**現狀**：兩個交易系統都在虧損，主要原因是：
1. **虧損加碼**（趨勢對齊 US30 加大 size 後巨虧）
2. **過度交易**（15M_趨勢跟隨一天做 13 筆 BTC）
3. **風報比倒掛**（多數策略贏小虧大，與系統設計目標「虧的快砍、賺的抱住」完全相反）
4. **策略畢業門檻太低**（全敗策略不該上線）
5. **Bug 導致績效數據不準**（43 筆 pnl=0）

**亮點**：
- 15M_RSI反轉（Forex）風報比極佳，是值得重點發展的方向
- mean_revert（Crypto）75% 勝率且正收益，是唯一可靠策略
- 系統架構（策略進化、風控暫停）設計合理，問題出在執行面

**建議方針**：先止血（停用全敗策略）→ 修 Bug → 收緊風控 → 擴大有效策略 → 開發新策略
